## Вопросы к бизнес-логике

- **Пачки (tracked_batch_sizes)** — поле хранится в `user_watchlist` ([10, 20, 30, 50] по умолчанию),
  но **нигде не используется в аналитике**. Алгоритм `find_best_batch_combination` принимает
  `target_quantity` из запроса, а не из watchlist.
  **Варианты решения:**
  1. Убрать поле как лишнее — пользователь сам вводит количество при запросе
  2. Реализовать группировку sell_options по этим размерам (например, "купить пачку 50 шт за X ₽")
  ⚠️ **Нужно решить до реализации страницы Склада.**

## UX — что переработать в интерфейсе

- **"Watchlist" → "Избранное"** — непонятен русскоязычному пользователю
- **ID товара** — скрыть из карточек, показывать только **название**
- **Категория товара** — перевести с английского (`weapon` → `Оружие`, `armor` → `Броня`)
- **Кнопка "Обновить"** — показывать таймер "Обновить через 1:43", не просто блокировать.
  Пояснение: данные обновляются автоматически каждые 5 мин, ручной refresh — дополнительная опция с кулдауном
- **Поиск лотов** — работать по названию, ID показывать только в деталях

## Дизайн — подсказки (Tooltips)

Везде где есть непонятный термин или цифра — добавить tooltip при наведении или иконку "?".

| Элемент | Текст подсказки |
|---------|-----------------|
| **Быстро** | Цена чуть ниже минимума на рынке. Продастся скорее, но заработаешь меньше. |
| **Нормально** | Рыночная цена. Баланс между скоростью и доходом. |
| **Выгодно** | Цена выше медианы. Придётся подождать, зато заработаешь больше. |
| **~3 ч / ~18 ч** | Прогноз: сколько обычно висит лот по такой цене до выкупа. |
| **Стабильный / Умеренный / Высокий риск** | Волатильность цены за 7 дней. Высокий риск = цена сильно скачет, прогноз ненадёжен. |
| **Лучшее время** | В этот час и день недели обычно больше всего покупок — выгоднее выставить лот заранее. |
| **Уверенность** | Насколько точен прогноз. Растёт по мере накопления данных о выкупах. |
| **Комиссия 5%** | Аукцион берёт 5% с каждой продажи. Уже учтено в расчёте прибыли. |
| **Ликвидных лотов** | Лоты с остатком времени ≥ 2ч. Лоты с меньшим остатком исключены — их цена нерыночная. |

**Важно:** заменить технические термины `Low / Medium / High` → `Стабильный / Умеренный / Высокий риск`. и добавить тултипы

## Система апрувов (доступ к сайту)

**Концепция:** закрытый портал — пользователи не могут войти без разрешения администратора.

**Логика:**
- При регистрации пользователь ждёт апрув (статус: pending)
- Администратор через панель выдаёт апрув на указанное количество дней (вводит вручную)
- После истечения срока — доступ закрывается, нужен новый апрув
- Без апрува: авторизация не проходит, даже если аккаунт существует

**Интерфейс admin-панели:**
- Строка поиска пользователей (по имени / email)
- Список пользователей с текущим статусом (pending / active / expired)
- Кнопка "Выдать апрув" → поле ввода количества дней → подтвердить
- Отображение: до когда действует апрув (дата истечения)

**Главная страница (лендинг для незалогиненных):**
- Короткое описание что такое портал и что он делает
- Кнопки "Войти" / "Зарегистрироваться"
- Пояснение что доступ по апруву (чтобы не было путаницы у новых пользователей)

⚠️ **Backend:** добавить поле `approved_until` (DateTime) в таблицу `users` + проверка при логине.
