# SC Trading — Контекст для Claude

## Блок 1: Управление контекстом

Одна задача = один чат. После завершения → /clear, открыть новый чат.  
История длинная в середине задачи → /compact, продолжить.  
В конце задачи (Шаг 6): сообщи «Задача выполнена. /clear».

## Блок 2: Алгоритм работы

| Задача | Читать |
|--------|--------|
| Что делать следующим | docs/NOTES.md |
| Формулы, расчёты | docs/BUSINESS_LOGIC.md |
| Сервисы, Celery задачи | docs/SERVICES.md |
| Таблицы БД, миграции | docs/DATABASE.md |
| Архитектура сбора данных | docs/ARCHITECTURE.md |

1. Читай `docs/NOTES.md` → первая `[ ]` задача
2. Читай **только** нужный docs/ файл по маршрутизатору выше
3. Покажи список файлов кода → **жди подтверждения**
4. Напиши код → объясни ЧТО и ПОЧЕМУ → **жди подтверждения** перед коммитом/пушем
5. Обнови только нужный docs/ (только diff): `[ ]` → `[x]`, метод/таблица/формула — только изменённое
6. «Задача выполнена. Документация обновлена: [список]. /clear»

## Блок 3: Правила коммуникации

До действия: «Планирую: [действие]. Затрону: [файлы]. Подтверждаешь?»  
Без подтверждения: не изменяй файлы, не коммить, не применяй миграции.  
После: «Сделано: [факт]. Изменено: [список файлов]»
