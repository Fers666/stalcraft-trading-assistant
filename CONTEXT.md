## Алгоритм работы

1. Читай `docs/NOTES.md` → бери первую `[ ]` задачу (одна задача = один чат)
2. По маршрутизатору — читай только нужный docs/ файл, не все сразу
3. Составь список файлов кода которые нужно трогать → покажи → жди подтверждения
4. Напиши/измени код → объясни что и почему → жди подтверждения перед коммитом/пушем
5. Обнови только нужный docs/ (только diff, не переписывай целиком)
6. Отметь `[ ]` → `[x]` в NOTES.md, новые задачи добавь в конец
7. Напиши: **«Задача выполнена. Документация обновлена: [список].»**

---

## Завершение чата

После завершения задачи — обязательно:

- История длинная в середине задачи → `/compact` (сжать, продолжить)
- Задача завершена → `/clear` затем открыть новый чат
- В новом чате написать: **«Открой NOTES.md и возьми следующую задачу»**

---

## Правила

- Всегда описывай что планируешь ДО того как делать: «Планирую: [действие]. Затрону: [файлы]. Подтверждаешь?»
- Без подтверждения: не изменяй файлы, не делай коммиты, не применяй миграции
- После выполнения: «Сделано: [факт]. Изменено: [список файлов]»
- Детали формул, таблиц, сервисов — в docs/, не здесь
