# SC Trading · design v5 — прототип «Терминал»

Автономный HTML/CSS/JS-прототип нового интерфейса SC Trading.
Работает **по file:// без сборки и сервера** — интернет нужен только для
Google Fonts (Inter, JetBrains Mono, Rajdhani) и иконок предметов (github EXBO).
Контракт дизайн-системы — `DIRECTION.md`, итоги этапа A — `AUDIT.md`.

## Статус

- **Этап A** — 3 направления на одной странице «Избранное»
  (`directions/a-terminal|b-hud|c-analytics`, обзор — `directions/index.html`).
  Победитель — **направление A «Терминал»**.
- **Этап B** — полное приложение из 9 страниц на системе направления A (`app/`).
- **QA-проход** — сквозная проверка консистентности выполнена (см. низ файла).

│   ├── data.js             реальные данные срезов (favorites, catalogExtra) — read-only
│   ├── app.js              SC_APP: форматтеры, сортировка, модалка, тост, armConfirm
│   ├── charts.js           SC_CHARTS: SVG-графики scatter/band (нужен только Избранному)

    ├── favorites.html      ЭТАЛОН: карточка предмета, лоты, графики, варианты продажи
    ├── catalog.html        каталог: дерево категорий, поиск, модалка добавления
    ├── lots.html           поиск лотов: автодополнение, фильтры (гейт: тариф ПРОДВИНУТАЯ+)

Порядок скриптов на страницах: `data.js → app.js → (charts.js только в favorites) → shell.js → страничный код`.

## Как смотреть

Открой `design/v5/app/index.html` двойным кликом (file://). Ничего запускать не нужно.
Демо-состояние тарифа живёт в `localStorage` (`sc_demo_tier`, `sc_demo_radar`) —
переключатель «демо» в навбаре меняет его без перезагрузки (событие `sc:tier`).

## Маршрут демо (рекомендованный порядок)

1. **Лендинг** (`index.html`) — hero с живым виджетом сигнала (крутится раз в 4 с,
   на hover — пауза), фичи на реальных данных, тарифы. «Войти» / «Начать бесплатно».
2. **Вход** (`login.html`) — поля уже заполнены, жми «Войти» → Избранное.
3. **Избранное** (`favorites.html`) — ядро продукта:
   - переключай предметы в правом сайдбаре и в ленте сигналов сверху;
   - сортируй таблицу выгодных лотов кликом по заголовкам; клик по строке
     пересчитывает «Варианты продажи»;
   - потыкай табы графика 24Ч/48Ч/7Д/30Д;
   - **смени тариф в демо-переключателе** (навбар): на БАЗОВАЯ появятся замки
     на 7Д/30Д, гейт с CTA поверх графика, «Лоты» в навбаре закроются,
     лимит избранного в сайдбаре сменится (6/10/20/25);
   - «Убрать» — двухшаговое подтверждение («Точно?», 3 с).
4. **Каталог** (`catalog.html`) — дерево категорий слева, поиск (Enter или «Найти»),
   пагинация; кнопка-закладка в строке открывает модалку добавления
   (регион; для артефактов — качество и заточка).
5. **Лоты** (`lots.html`) — автодополнение от 2 символов (стрелки + Enter),
   история запросов, фильтры качество/заточка, «строк на страницу»;
   на тарифе ниже ПРОДВИНУТАЯ+ — полностраничный замок с CTA.
6. **Радар** (`radar.html`) — рейтинг «за чем следят трейдеры»;
   **выключи чекбокс «радар»** в демо-переключателе — полностраничный замок аддона,
   «Включить в демо» возвращает.
7. **Склад** (`inventory.html`) — «Добавить товар» (поиск в модалке, кол-во, цена),
   P&L по каждой позиции и по складу; удаление — двухшаговое.
8. **Настройки** (`settings.html`) — критерий выгодности (живая связь с инфобоксом
   Telegram), «Получить код привязки» (таймер 10:00, копирование),
   «(демо) Симулировать привязку» включает тумблер уведомлений; панель тарифа
   реагирует на демо-переключатель. Вход со страниц — шестерёнка в навбаре (подсвечена).
9. **Лента** (`feed.html`) — чипы-фильтры типов событий, «Мин. профит»,
   «Только избранное»; сводка 24ч и «Лучший сигнал дня» в сайдбаре;
   раз в 45 с приходит новое live-событие. Кнопки «Лоты»/«Карточка» ведут
   на страницы с выбранным предметом.

Тупиков нет: заглушки («Новости», «Помощь», FAQ, «Смотреть тарифы») отвечают
тостом «вне охвата прототипа», а не мёртвым переходом.

## Что дальше

Спека внедрения готова: `docs/tasks/design-v5-implementation.md` (7 фаз, независимый деплой; фаза 7 «Лента» — после бэкенд-обсуждения).

1. `researcher`/`designer` — спека переноса (маппинг компонентов прототипа на
   `frontend/src`: theme.ts-токены, компоненты страниц, стор гейтинга тарифов).
2. `frontend-dev` — имплементация по спеке (React + MUI, навбар остаётся плоским `div`).

Известные упрощения прототипа (не баги): данные детерминированно «доращиваются»
от реальных срезов (каталог ×35, лоты вне избранного — от категорийных медиан);
словари категорий (`CAT_RU`, `TREE`) продублированы в catalog/lots — в продукте
это один модуль (`frontend/src/utils/i18n.ts`, `categories.ts`).
