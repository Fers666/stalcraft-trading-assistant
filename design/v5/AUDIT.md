# Дизайн-аудит фронтенда SC Trading — 2026-07

Аудируемая поверхность: `frontend/src/` — глобальный слой (theme, Layout, GlobalFeed, EmissionWidget, LotStatCard, PriceChart, SalesHistoryCharts) и 9 страниц (Landing, Login, Monitoring, Catalog, Lots, MarketRadar, Inventory, Settings, Feed). RegisterPage/NewsPage/FaqPage/AdminPage — вне охвата (NewsPage упомянут только как носитель третьего паттерна удаления).

Контекст продукта: «Bloomberg-терминал для Зоны» — pro-инструмент трейдера (ANOMALY → DATA → PROFIT). Регистр — **product** (design serves the task): ценится плотность данных, скорость чтения цифр, earned familiarity, консистентность экранов. Итерация «Магма»: золото #B78A2A/#D9AF37/#F2C94C/#FFB800 на #080808, поверхности #11151A/#1A1F26/#202633, Rajdhani + Inter, MUI.

---

## Методология

Комбинация трёх чек-листов:

1. **Эвристики Нильсена** — консистентность и стандарты (H4), видимость статуса системы (H1), предотвращение ошибок (H5), узнавание vs припоминание (H6), эстетика и минимализм (H8).
2. **impeccable / audit + product register** — 5 измерений (A11y, Performance, Theming, Responsive, Anti-Patterns) с оценкой 0–4; продуктовые баны: непоследовательный словарь компонентов, спиннеры вместо скелетов, декоративный motion, display-шрифт в данных.
3. **redesign-skill** — поиск generic-AI-паттернов: смешение тёплых/холодных серых, больше одного акцента, сырые Material-цвета, `confirm()`, отсутствие hover/focus-состояний, inline-стили вперемешку с системой стилей, произвольные z-index.

### Оценка здоровья (impeccable audit health score)

| # | Измерение | Оценка | Ключевая находка |
|---|-----------|--------|------------------|
| 1 | Accessibility | 1/4 | Кликабельные `div`/`Typography` без клавиатуры и focus-стилей; контраст #7C7C7C и #555 на карточках ниже 4.5:1 при кеглях 8–10px |
| 2 | Performance | 3/4 | IntersectionObserver-ленивая загрузка графиков — хорошо; `transition: all`, ремаунт FiltersBar |
| 3 | Responsive | 2/4 | Desktop-инструмент (допустимо), но три разные ширины сайдбаров и захардкоженный sticky-offset 156px |
| 4 | Theming | 2/4 | Токены есть, но 3 конкурирующие системы цветов качества, off-palette красные/зелёные, inline-стили мимо темы |
| 5 | Anti-Patterns | 2/4 | Gradient text и numbered eyebrows на лендинге, `confirm()`, сырые Material-цвета — при этом общий облик «Магмы» самобытный |
| **Итого** | | **10/20** | **Acceptable — нужна системная работа, не точечные фиксы** |

**Вердикт по анти-паттернам:** приложение НЕ выглядит AI-generated — золотая идентика, алмаз-логотип, терминальные киккеры и плотность данных дают собственный характер. Проблема другая: **три поколения кода живут одновременно** (страницы до дизайн-системы, страницы «Магмы», мёртвый Navbar), и швы между ними видны пользователю — это ломает доверие к данным сильнее, чем любой отдельный визуальный недочёт.

## Приоритеты

- **P0** — ломает доверие к данным или базовую юзабилити; чинить в первую очередь.
- **P1** — заметная несогласованность или нарушение WCAG AA; чинить до релиза v5.
- **P2** — раздражение, есть обходной путь; в рамках редизайна.
- **P3** — полировка; по остаточному принципу.

## 1. Глобальный слой

### 1.1 Цвет и токены

**Что хорошо:** `theme.ts` — настоящая дизайн-система: палитра названа (G1–G4, BG0–BG3, T0–T2), статусные цвета определены, MUI-оверрайды покрывают 20+ компонентов, `tokens` экспортированы для прямого использования. Дублирующие CSS-переменные в `frontend/index.html:14-31` синхронизированы с theme.ts.

**Находки:**

- **[P0] Три конкурирующие системы цветов качества предметов.** Один и тот же предмет качества «Легендарный» — **золотой #FFB800** в Избранном/ленте (`frontend/src/utils/i18n.ts:74-81`), но **красный #f44336** в Каталоге (`frontend/src/pages/CatalogPage.tsx:51-69`, ключи по `color` из БД) и в Лотах (`frontend/src/pages/LotsPage.tsx:98-105`, ключи по русскому имени). «Особый» — голубой #53B7FF против синего #2196f3. Для трейдера цвет качества — семантика цены; расхождение между экранами читается как баг данных. Плюс CatalogPage/LotsPage используют сырую Material-палитру (#4caf50/#2196f3/#9c27b0/#ff9800/#f44336) — фиолетовый и синий запрещены дизайн-системой. → Единый словарь `QUALITY_COLORS` в theme.ts, построенный на статусной палитре токенов.
- **[P1] Второй зелёный.** `frontend/src/pages/MonitoringPage.tsx:288,289,298,319,321` — маркер «выгодный» жёстко `#4caf50` (Material green), при том что `tokens.success = #3ED598`. Два зелёных в одном сайдбаре, GlobalFeed при этом использует правильный `success.main`.
- **[P1] Второй и третий красный.** `frontend/src/components/EmissionWidget.tsx:42-57` — Tailwind-красные `#DC2626`/`#EF4444` вместо `tokens.danger = #FF5A5A`. Там же `borderRadius: 4` против системных 8/12.
- **[P2] Легаси purple*-алиасы.** `frontend/src/theme.ts:450` — `purple: G2, purpleBright: G3, purpleSoft: G3, purpleDark: G1`. Мёртвый словарь из до-золотой эры; провоцирует `tokens.purple` в новом коде. → удалить после grep по использованиям.
- **[P2] Rgba-литералы вместо токенов.** Десятки мест: `rgba(217,175,55,…)` (MonitoringPage.tsx:292,294; GlobalFeed.tsx:32,41), `rgba(62,213,152,…)` (GlobalFeed.tsx:151-158), `rgba(255,255,255,0.02…0.08)` повсюду. Тот же цвет — то `alpha(tokens.gold, 0.12)`, то `'rgba(217,175,55,0.12)'`. → зафиксировать альфа-шкалу в токенах (gold08, gold12, line, lineStrong…).
- **[P2] Смешение inline `style={}` и `sx`+tokens — нет единого источника.** `frontend/src/components/Layout.tsx:61-245` — весь навбар на inline-стилях с хардкодами `#F5F5F5` (:99), `#7C7C7C` (:103), `#B8B8B8` (:152,165); `frontend/src/components/EmissionWidget.tsx:31-49` — inline `React.CSSProperties`. Рядом те же значения берутся из `tokens` через `sx`. Inline-стили не видят тему, ломают будущую темизацию и утяжеляют DOM.
- **[P3] Telegram-синий #229ED9** (`frontend/src/pages/SettingsPage.tsx:219,269-271,287`) — допустимое брендовое исключение, но стоит вынести в токен `brandTelegram`, чтобы исключение было именованным.

### 1.2 Типографика

**Что хорошо:** пара Rajdhani (display) + Inter (UI/данные) — контрастная ось «узкий технический display + нейтральный гротеск», это правильный тип пары. Letter-spacing киккеров задаёт терминальный голос.

**Находки:**

- **[P1] Цифры без табулярной моноширинности.** Ни одного `font-variant-numeric: tabular-nums` в проекте. Цены — главный контент («скорость чтения цифр»), а в Inter пропорциональные цифры: колонки цен в LotStatCard/LotsTable «дышат» по ширине, разряды не выравниваются, сравнение глазами замедляется. Местами вместо этого `fontFamily: 'monospace'` — системный fallback без контроля начертания (`frontend/src/components/LotStatCard.tsx:357,729`, `frontend/src/pages/InventoryPage.tsx:125`, `frontend/src/pages/MarketRadarPage.tsx:190`). → единый numeric-стиль: Inter + tabular-nums (или выделенный мономейс для данных), выравнивание цен по правому краю везде.
- **[P1] Зоопарк микро-кеглей ниже порога читаемости.** Ad-hoc размеры 0.5rem (8px), 0.52, 0.54, 0.55, 0.58, 0.6, 0.62, 0.65, 0.68, 0.72rem… Примеры: подпись логотипа 8px (`frontend/src/components/Layout.tsx:103`), таймштамп ленты 0.52rem (`frontend/src/components/GlobalFeed.tsx:51`), время лота 0.54rem (GlobalFeed.tsx:192), лейблы «МЕДИАНА 7Д» 0.55rem (`frontend/src/components/LotStatCard.tsx:415`), «за 7д» 0.5rem (`frontend/src/pages/MarketRadarPage.tsx:217`). 8–9px — нечитаемо и не проходит WCAG в паре с muted-цветом (см. 1.8). → шкала с полом: данные ≥ 12px, вспомогательный текст ≥ 11px, декоративные киккеры ≥ 10px.
- **[P2] Rajdhani просачивается в данные.** Аватар-фоллбек списка (`frontend/src/pages/MonitoringPage.tsx:309`), ранг в радаре (`frontend/src/pages/MarketRadarPage.tsx:144`) — по product-регистру display-шрифт в данных запрещён; ранг топа — допустимая «витрина», буква предмета — нет.
- **[P3] `!important` в типографике кнопки.** `frontend/src/theme.ts:161` — `color: '#080808 !important'` для containedPrimary. Контраст тёмного текста на золоте корректен (≈9:1; светлый #F5F5F5 дал бы ~1.9:1 — менять нельзя), но `!important` — признак войны специфичности, лучше решить через слоты.

### 1.3 Spacing и сетка

- **[P2] Три ширины сайдбара на трёх страницах:** 260px (`frontend/src/pages/MonitoringPage.tsx:190`), 230px (`frontend/src/pages/CatalogPage.tsx:216`), 210px (`frontend/src/pages/LotsPage.tsx:492`). Одинаковая роль (навигация по списку/дереву) — три разных размера, при переходах между страницами колонка «прыгает».
- **[P2] Магический sticky-offset.** `frontend/src/pages/MonitoringPage.tsx:196` — `top: '156px'` (56 навбар + 84 лента + отступ), но лента условная: без неё сайдбар липнет со 100px мёртвого зазора. Layout при этом честно считает `topOffset` динамически (`frontend/src/components/Layout.tsx:253`). У LotsPage тот же паттерн — `top: 16` (`frontend/src/pages/LotsPage.tsx:501`). → пробросить offset из Layout (CSS-переменная).
- **[P3] Колонки прибыли 86px** захардкожены в двух местах (`frontend/src/components/LotStatCard.tsx:303,659`) — при цене >1 млн «итого +1 234 567 ₽» упирается в границу.

### 1.4 Иконография

- **[P2] Два конкурирующих набора.** Рабочий навбар — кастомные stroke-SVG 14px (`frontend/src/components/Layout.tsx:15-48`), характерные и единые по стилю (+ повторно нарисованный LockIcon в 3 файлах: LotStatCard.tsx:14-18, SalesHistoryCharts.tsx:28-32, Layout.tsx:27,47). Страницы — Material Icons (filled, другая пластика): Search/Delete/Add/Bookmark/Trending… Смешение двух языков в одном экране (LotStatCard: кастомный замок + Material TrendingUp/Search/Delete). → выбрать один язык (кастомный stroke-набор ближе к идентике), вынести LockIcon в общий компонент.
- **[P3] Стрелки сортировки тремя способами:** текстовые ` ▲`/` ▼` (LotStatCard.tsx:540,556), Material ArrowUpward/Downward (LotsPage.tsx:396-401), ▲/▼ в чипах времени (LotStatCard.tsx:461,466 — там это «продажа/покупка», конфликт значений с сортировкой).

### 1.5 Motion

- **[P2] `transition: 'all 0.2s'`** — в теме кнопок (`frontend/src/theme.ts:154`) и навбара (`frontend/src/components/Layout.tsx:159,190`). Анимируется всё подряд, включая layout-свойства. → перечислять свойства (color, background-color, border-color, transform).
- **[P3] Нет `prefers-reduced-motion`** ни для `anomaly-pulse` (`frontend/index.html:50-53`), ни для transition ленты.
- **[P3] Живые данные не анимированы.** Данные LotStatCard/ленты обновляются каждые 30с (`frontend/src/components/LotStatCard.tsx:182`) молча: цена может смениться под курсором без индикации. Для терминала обновление — событие: короткий flash/tint изменившейся ячейки — стандарт Bloomberg-класса. Единственная motion-идея сейчас — hover-подъём карточек ленты (GlobalFeed.tsx:154-160), хорошая.

### 1.6 Плотность данных и состояния

**Что хорошо:** LotStatCard — образец правильной плотности: заголовок с медианой, статус-бар, таблица выгодных лотов с сортировкой, варианты продажи с пересчётом от выбранного лота (:567-611 — клик по лоту, подсказка «↳ Кликните лот…»), гистограмма пачек. GlobalFeed со скелетоном при загрузке (GlobalFeed.tsx:99-116) — единственное место со скелетоном, и это правильный паттерн.

- **[P2] Спиннеры вместо скелетонов** во всех остальных местах: LotStatCard.tsx:325-329 (CircularProgress в пустоте 300px — карточка «мигает» при каждой смене предмета), MonitoringPage.tsx:139, CatalogPage.tsx:290, LotsPage.tsx:658, InventoryPage.tsx:87, MarketRadarPage.tsx:65. Product-регистр: скелет, повторяющий макет.
- **[P2] Locked-состояния тарифов — 4 разных вида:** тултип на сером div с замком (SalesHistoryCharts.tsx:58-67), замок+chip «30д» (LotStatCard.tsx:388-397), «Недоступно на тарифе» по центру (LotStatCard.tsx:621-636), заблокированный пункт навбара (Layout.tsx:120-141). Ни один не даёт CTA «как получить тариф». → единый компонент LockedFeature с одинаковым текстом и действием.

### 1.7 Нейминг и голос

- **[P2] Три имени продукта у пользователя:** «SC TRADING» (навбар `frontend/src/components/Layout.tsx:101`, `frontend/index.html:7`), «STALZONE» как имя аукциона (`frontend/src/pages/LandingPage.tsx:226`, `frontend/src/pages/LoginPage.tsx:101`, FaqPage), «SZ Trading Assistant» в Telegram-боте и заголовке API (`backend/app/main.py:29`, `backend/app/api/v1/endpoints/telegram.py:151`). CHANGELOG фиксирует различие как осознанное, но пользователь, пришедший из бота в веб, видит три бренда. → одна связка: продукт «SC Trading», площадка «аукцион STALZONE», бот — «SC Trading Bot».
- **[P2] «ты» и «вы» вперемешку:** «Получишь (−5%)» (`frontend/src/components/LotStatCard.tsx:676`), «Зарегистрируйся» (LoginPage.tsx:103) — против «Вы сможете добавить его снова» (`frontend/src/pages/MonitoringPage.tsx:374`), «обратитесь к администратору» (MarketRadarPage.tsx:76). Для аудитории 14–25 выбрать «ты» и выровнять.
- **[P3] Англоязычные киккеры в русском UI:** «ITEM DATABASE // 2 236+ ENTRIES» (`frontend/src/pages/CatalogPage.tsx:180`), «ZONE MARKET TERMINAL» — как элемент терминального стиля работает, но нужно правило: EN — только декоративные киккеры, никогда — носители смысла (при этом «2 236+» захардкожен и устареет).

### 1.8 Доступность

- **[P1] Кликабельные не-интерактивные элементы без клавиатуры и фокуса.** Карточки ленты — `Box onClick` (`frontend/src/components/GlobalFeed.tsx:143-161`), строки выгодных лотов — `Box onClick` (`frontend/src/components/LotStatCard.tsx:570-580`), сортировка — `Typography onClick` (LotStatCard.tsx:531-541,549), логотип — `div onClick` (`frontend/src/components/Layout.tsx:73-75`), заголовки таблицы — `TableCell onClick` без `aria-sort` (`frontend/src/pages/LotsPage.tsx:824-847`). Tab до них не доходит, Enter не работает, focus-стиля нет.
- **[P1] Нет видимых focus-стилей у главной навигации.** NavLink стилизован только инлайном по `isActive` (`frontend/src/components/Layout.tsx:144-170`) — состояние `:focus-visible` не определено нигде в проекте (grep по `focus-visible` — пусто).
- **[P1] Контраст muted-текста.** `#7C7C7C` на карточке `#1A1F26` ≈ **4.06:1** — ниже 4.5:1, и этим цветом набраны 8–10px лейблы (двойное нарушение: LotStatCard.tsx:415,449; MarketRadarPage.tsx:198,213). Чип «Обычный» `#555` на `#1A1F26` ≈ **2.3:1** (`frontend/src/pages/LotsPage.tsx:99`, `frontend/src/pages/CatalogPage.tsx:52`). «СИГНАЛЫ» rgba(217,175,55,0.65) при 0.58rem (GlobalFeed.tsx:40-42). → поднять muted до ~#9A9A9A для мелких кеглей или укрупнить текст.
- **[P2] `Avatar src` без alt** во всех списках (иконки предметов — значимые изображения); icon-only IconButton без `aria-label` (Tooltip даёт description, не name).
- **[P3] Иерархия заголовков начинается с h5** (`variant="h5"` как заголовок страницы везде) — нет h1; для SPA-терминала не критично, но выравнивается бесплатно через `component="h1"`.
- **[P3] z-index россыпью:** 1300 (Layout.tsx:69), 1200 (GlobalFeed.tsx:16), 10 (LotsPage.tsx:588), 1 (LandingPage) — работает на MUI-шкале, но своя семантическая шкала в токенах надёжнее.

## 4. Выводы: принципы редизайна для design/v5

Сформулированы как инварианты: обязательны для всех трёх дизайн-направлений (Терминал / Тактический HUD / Pro-аналитика soft-dark); направления различаются поверх них — плотностью, фактурой, температурой поверхностей и характером motion.

**1. Один источник цветовой правды.** Каждый цвет в UI существует только как токен theme.ts: золотая шкала, поверхности, статусы — и новая единая шкала качества предметов (сейчас «Легендарный» — золотой или красный в зависимости от страницы). Удаляются purple*-алиасы, локальные словари цветов, Tailwind/Material-литералы и inline-стили с хексами. Правило ревью: хекс вне theme.ts = дефект.

**2. Цифры — главный контент.** Все цены и метрики — tabular-nums (или выделенный data-шрифт), выровнены по правому краю, с единым форматированием «1 234 567 ₽». Иерархия чтения: значение > дельта/прибыль > метаданные. Rajdhani — только заголовки и брендовые киккеры, никогда — данные. Изменение значения при live-обновлении — видимое событие (короткий flash ячейки), а не молчаливая подмена.

**3. Типографская шкала вместо зоопарка.** Фиксированная шкала (например 10/11/12/13/14/16/20/28px) с жёстким полом: данные и текст ≥ 12px, вспомогательное ≥ 11px, декоративные киккеры ≥ 10px. Muted-цвет пересчитывается под контраст ≥ 4.5:1 на самой светлой поверхности, где используется (сейчас #7C7C7C на #1A1F26 ≈ 4.06:1, а #555-чипы ≈ 2.3:1).

**4. Единый словарь компонентов.** Одно действие = один паттерн на всех экранах: удаление (двухшаговая inline-кнопка для строк; диалог только при потере невосстановимых данных; `confirm()` запрещён), выбор региона (Select из общих констант), добавление в избранное (одна кнопка-иконка), locked-состояние тарифа (один компонент с CTA), сортировка (один индикатор), уведомление об успехе (один канал). Мёртвый Navbar.tsx и оверрайды MuiAppBar удаляются — канон один: div-навбар Layout.

**5. Состояния — граждане первого класса.** Каждая загрузка — скелетон формы контента (эталон уже есть: GlobalFeed), каждый empty-state учит следующему шагу (CTA), каждый интерактивный элемент — настоящий button/link с hover, active, disabled и видимым `:focus-visible` (золотое кольцо). Тупиков в главной навигации нет: «Лента» либо скрыта до готовности, либо ведёт к действию.

**6. Плотность по сетке, а не по случаю.** Базовая сетка 4px; сайдбары одной ширины и одного поведения; sticky-offset вычисляется из реальной высоты навбар+лента (CSS-переменная из Layout), а не магических 156px; таблицы плотные, панели дышат — плотность является осознанным параметром направления, но геометрия каркаса едина.

**7. Motion передаёт состояние, бренд — одним голосом.** Транзишены 150–250ms ease-out по перечисленным свойствам (не `all`), `prefers-reduced-motion` обязателен, никакой оркестровки при загрузке; допустимая декорация — только брендовые живые элементы (пульс статуса, лента сигналов). Нейминг един: продукт «SC Trading», площадка «аукцион STALZONE», бот и API — под брендом SC Trading; обращение к пользователю — «ты» везде.
